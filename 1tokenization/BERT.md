BERT = Bidirectional Encoder Representations from Transformers（基于 Transformer 的双向编码器表示）
核心思想：双向理解
之前的方法只能单向读句子：
单向（从左到右）:  "我 在 梦 里 收 到 清华大学 录取 通知书"
                   → 读到"清华大学"时，还不知道后面是"录取通知书"
BERT 的创新是双向同时看上下文：
BERT（双向）:  "我 在 梦 里 收 到 清华大学 录取 通知书"
                 ← —————————————— 清华大学 ——————————————→
                 能同时看到"收到"和"录取"，理解更准确

BERT 是怎么训练的？（两个任务）
1. 掩码语言模型（Masked Language Model, MLM）

随机遮住一些词，让模型猜：
原文:  "我在梦里收到 [MASK] 录取通知书"
输入:  "我在梦里收到 [MASK] 录取通知书"
      ┌─────────────────────────────────────┐
      │              BERT                    │
      └──────────┬──────────────────────────┘
                 ↓
答案:  "清华大学" ✓

这就像你做完形填空一样。

2. 下一句预测（Next Sentence Prediction, NSP）

判断两句话是否相邻：
句子A: "我在梦里收到清华大学录取通知书"
句子B: "我高兴得跳了起来"
      → 预测: 是/否？  ✓ 是

句子A: "我在梦里收到清华大学录取通知书"  
句子B: "今天天气很热"
      → 预测: 是/否？  ✗ 否

BERT 的架构
┌─────────────────────────────────────┐
│          BERT 模型结构               │
├─────────────────────────────────────┤
│  12 层 Transformer Encoder 堆叠      │  ← 只用编码器，不用解码器
│  每层包含:                           │
│    - Multi-Head Self-Attention      │  ← 关键！让每个词都看到其他词
│    - Feed-Forward Network           │
│  隐藏层维度: 768                     │
│  注意力头数: 12                      │
├─────────────────────────────────────┤
│  词表大小: 30,000 (中文约21,000字)   │  ← 这就是你 tokenizer 用到的词表
└─────────────────────────────────────┘
